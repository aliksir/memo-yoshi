# MEMO-YOSHI — Claude Codeへの付箋

人間が雑に書いたメモをClaude Codeが読み書き・整理・アクション実行するスキル。

## 対応環境

Claude Code と Claude Cowork の両方で動作する。Bashが使えない環境（Cowork等）では、ブラウザ起動やコマンド実行を代替案内に自動切替する。

## メモファイルの場所

以下の順序で探す:
1. カレントディレクトリの `MEMO-YOSHI.md`
2. `~/.claude/MEMO-YOSHI.md`

見つからない場合は `~/.claude/MEMO-YOSHI.md` を新規作成する。

## サブコマンド判定

引数 `$ARGUMENTS` を見てサブコマンドを判定する:

| 引数 | サブコマンド |
|------|------------|
| なし / `open` | **open** — エディタで開く |
| `read` | **read** — 内容を読み込む |
| `tidy` | **tidy** — 整理・カテゴリ分け |
| `add ...` | **add** — テキストを追記 |
| `sync` | **sync** — memory-mcp ↔ MEMORY.md の同期チェック・相互補完 |
| `act` | **act** — 中身を解析してアクション実行 |
| `done` / `done ...` | **done** — 指定項目 or 直前のact完了項目に「終わりヨシッ！」マークをつける |
| それ以外のテキスト | **add** として扱う（引数全体をメモとして追記） |

## 各サブコマンドの手順

### open（デフォルト）

1. メモファイルのパスを特定する
2. **Claude Code**: Bashツールで OS に応じたコマンドでファイルを開く:
   - Windows: `start "" "パス"`
   - macOS: `open "パス"`
   - Linux: `xdg-open "パス"`
3. **Claude Cowork（Bashなし）**: ファイルパスを表示して手動で開くよう案内する:
   - 「MEMO-YOSHI.md のパスは `{パス}` です。テキストエディタで開いてください」
4. 「MEMO-YOSHI.md を開いたヨシッ！」と報告

### read

1. メモファイルを Read ツールで読み込む
2. 内容を要約してユーザーに提示する:
   - メモの件数（箇条書きの数 or セクション数）
   - 主なトピック
   - 対応が必要そうな項目があればハイライト
3. 「読み込み完了。N件のメモがあるヨシッ！」と報告

### tidy

1. メモファイルを Read ツールで読み込む
2. **裸URL検出・解決**（tidyの最重要機能）:
   - 裸URLの定義: 行がURL（https://...）のみで、説明テキスト・日付・マークが一切ない行
   - 例: `https://x.com/user/status/123456` ← これが裸URL
   - 非例: `- https://... → ツール名。説明文 (2026-03-17)` ← これは整理済み
   - 各裸URLに対して:
     a. playwright-browser MCPツール（browser_navigate → browser_get_html `[data-testid='tweetText']` or article）で内容を取得
     b. X投稿の場合: 投稿者名・本文テキストを抽出
     c. GitHub/その他の場合: ページタイトル・要約を抽出
     d. 取得した内容を1行で要約し、以下の形式に変換:
        `- {要約} {URL} (YYYY-MM-DD)`
     e. playwright-browserが使えない場合はWebFetchでフォールバック
   - **分類先の判定**:
     - ツール・ライブラリ・リポジトリ → `## アイデア・メモ` セクション
     - セキュリティ・脆弱性情報 → `## URL・リンク` セクション
     - タスク・やりたいこと → `## TODO・依頼` セクション
     - その他情報系 → `## URL・リンク` セクション
3. 内容を分析して以下のカテゴリに整理:
   - **URL・リンク**: URLが含まれるメモ
   - **TODO・依頼**: 「〜して」「〜したい」「TODO」等のアクション項目
   - **アイデア・メモ**: 上記以外の自由メモ
   - **済み**: 完了マーク付き（`[x]` や `~~取り消し線~~`）
4. カテゴリごとにセクション分けして Edit ツールで書き戻す
5. 元の内容を消さない。構造化するだけ（裸URLは要約付きエントリに変換）
6. 「整理完了。URL解決: N件、URL: N件、TODO: N件、メモ: N件 ヨシッ！」と報告

### add

1. `$ARGUMENTS` から `add` プレフィックスを除いたテキストを取得
2. メモファイルの末尾に追記（Edit ツール）:
   ```
   - {テキスト} (YYYY-MM-DD HH:MM)
   ```
3. 「メモ追加したヨシッ！」と報告

### act

1. メモファイルを Read ツールで読み込む
2. 各メモ項目を分析して、実行可能なアクションを特定:

   | パターン | Claude Code | Cowork |
   |---------|------------|--------|
   | URL（http/https） | Bashでブラウザを開く | URLを表示して手動で開くよう案内 |
   | 「〜を調べて」「リサーチ」 | WebSearch で調査→書き戻す | WebSearch で調査→書き戻す |
   | 「〜を確認して」「バージョン確認」 | Bash実行→書き戻す | 実行不可。⏳マークで案内 |
   | 「〜を直して」「〜を実装して」 | **実行しない**。確認を求める | **実行しない**。確認を求める |
   | ファイルパス | ファイル存在確認→要約 | ファイル存在確認→要約 |

3. 実行結果を各メモ項目の横に追記（日時付き）:
   ```
   - https://example.com → ✅ 開いた (2026-03-14 12:00)
   - Node.jsのバージョン確認 → ✅ v22.5.0 (2026-03-14 12:01)
   - バグ修正して → ⏳ 依頼として認識。実行しますか？
   ```
4. act完了後、✅ がついた項目には自動で `終わりヨシッ！` マークをつける
5. 「N件のアクションを実行したヨシッ！（M件は確認待ち）」と報告

### done

完了した項目に `終わりヨシッ！` マークをつける。マーク付きの項目は read / act 時にスキップされる。

1. 引数がある場合（`/memo done Node.jsの...`）: 該当するメモ項目を検索
2. 引数がない場合（`/memo done`）: ✅ マークがついている全項目を対象
3. 対象項目の行頭に `終わりヨシッ！` を追加:
   ```
   - 終わりヨシッ！ Node.jsのバージョン確認 → ✅ v24.13.1 (2026-03-14 12:45)
   ```
4. 「N件を終わりヨシッ！にしたヨシッ！」と報告

### sync

memory-mcp（ナレッジグラフ）と MEMORY.md の整合性チェック・相互補完を行う。

**前提**: 2つのメモリストアの役割分担:
- **MEMORY.md**: 運用設定・PJリスト・ルールポインタ（手動管理、毎ターン注入）
- **memory-mcp**: 教訓・研究知見・TODO・外部リファレンス（蓄積型、検索向き）

**手順**:
1. `mcp__memory__read_graph` で memory-mcp の全エンティティを取得
2. MEMORY.md を Read で読み込む
3. 以下の4つのチェックを実行:

   **チェック1: memory-mcpの鮮度**
   - 各エンティティの observation に含まれるファイルパスが実在するか Glob/ls で確認
   - 実在しないパスを参照しているエンティティを「陳腐化」として報告

   **チェック2: MEMORY.md → memory-mcp の欠落**
   - MEMORY.md の「Feedback」「参照ファイル」セクションの各エントリが memory-mcp にも存在するか確認
   - 存在しないものを `mcp__memory__add_observations` or `mcp__memory__create_entities` で追加提案

   **チェック3: memory-mcp → MEMORY.md の欠落**
   - memory-mcp に重要なエンティティがあるのに MEMORY.md に言及がないものを検出
   - 特に新しい Lesson や TODO で MEMORY.md の「参照ファイル」に未掲載のもの

   **チェック4: MEMORY.md行数チェック**
   - 現在の行数と200行上限を比較
   - 上限に近い場合、memory-mcpに移動可能な項目を提案（将来タスクリスト等）

4. チェック結果をまとめて報告:
   ```
   sync結果:
   - 陳腐化エンティティ: N件
   - MEMORY.md→memory-mcp欠落: N件
   - memory-mcp→MEMORY.md欠落: N件
   - MEMORY.md行数: N/200行
   修正提案: ...
   ```
5. 修正は**提案のみ**。実行はユーザー承認後
6. 「同期チェック完了 ヨシッ！」と報告

## 終わりヨシッ！マークの扱い

- `終わりヨシッ！` が行頭についた項目は**完了済み**
- **read**: `終わりヨシッ！` 付き項目はスキップする。件数だけ「他N件は完了済み」と報告
- **act**: `終わりヨシッ！` 付き項目はスキップする（再実行しない）
- **tidy**: `終わりヨシッ！` 付き項目は「## 終わりヨシッ！」セクションにまとめる
- ユーザーが「全部読んで」「完了済みも含めて」と指定した場合のみ、全項目を表示する

## 重要なルール

- **メモを勝手に削除しない**。tidyは整理（構造化）のみ。削除はユーザーの指示で行う
- **actで破壊的操作をしない**。URLを開く・調べる等の安全な操作のみ自動実行。実装・修正系は確認を求める
- メモファイルが存在しない場合は `~/.claude/MEMO-YOSHI.md` にテンプレートを作成してから操作する

## Gotchas

- **ファイル探索順序に注意**: カレントディレクトリの`MEMO-YOSHI.md` → `~/.claude/MEMO-YOSHI.md`の順で探索。両方に存在する場合、カレントディレクトリが優先される
- **actで破壊的操作を実行しない**: URLを開く・調べる等の安全な操作のみ自動実行。「〜を直して」「〜を実装して」はユーザー確認を求める
- **終わりヨシッ！マーク付き項目はスキップされる**: read/actで完了済み項目は自動スキップ。全項目を見たい場合はユーザーが明示的に指定する
